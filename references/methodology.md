# Methodology / 方法说明

本 Agent 用 Pandadata 的融资余额、龙虎榜热度和成交金额线索，观察热门交易是否已经进入拥挤状态。它适合用于风险预警、抱团交易复盘和去杠杆风险观察。

## 核心问题

热门交易是否已经拥挤到需要提高风险警惕？

## 使用的数据

- Pandadata methods: `get_margin`, `get_lhb_list`
- 主要产物：`outputs/live/crowding_scorecard.json`
- 关键图表：
  - `outputs/live/charts/funding_balance_crowding.png`
  - `outputs/live/charts/lhb_amount_top.png`
  - `outputs/live/charts/scorecard_metrics.png`

## 判断逻辑

1. 观察融资余额是否扩张。
2. 统计龙虎榜事件热度和高金额样本。
3. 检查热度是否集中在少数标的，还是分散到更多样本。
4. 当融资扩张、龙虎榜热度高、成交金额集中同时出现时，拥挤风险上升。
5. 当价格承接转弱但杠杆和热度仍高时，需要重点观察踩踏和去杠杆风险。

## 输出解释

- `拥挤观察`：热门交易热度较高，风险预算应同步收紧。
- `常规关注`：当前证据不足以说明交易已经明显拥挤。
- `融资余额变化`：杠杆资金是否继续参与热门交易。
- `龙虎榜热度`：事件样本密度和资金关注度。

## 适合继续追问的问题

- 高金额样本是否集中在少数标的？
- 次日承接是否还能支撑热度？
- 融资扩张是否与价格转弱同时出现？
- 这类拥挤是机会扩散，还是踩踏前兆？

## 使用边界

拥挤风险是风险状态，不等于价格一定反转。本 Agent 不做方向预测，也不替代用户自己的仓位和风控规则。
